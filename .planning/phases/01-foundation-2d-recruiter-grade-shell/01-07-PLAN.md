<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `public/assets/*` ↔ Public web | Every byte under `public/assets/` becomes publicly downloadable on `eren-atalay.github.io`; OPSEC gate must catch metadata leaks before upload. |
| Source code ↔ Production bundle | The text of `src/**/*.tsx` and the contents of `index.html` end up in `dist/` verbatim (modulo Vite minification); anything sensitive committed to source becomes shipped to recruiters. |
| External link from site ↔ Third-party origin | `target="_blank"` links without `rel="noopener noreferrer"` leak referrer + window.opener access (tabnabbing). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | EXIF / GPS / Author / CreatorTool metadata in `public/assets/*` (CV PDF, OG image, future screenshots) | mitigate | Plan 02's `[CI]` exiftool strip + verify gate enforces on every deploy. This plan's checklist documents the gate AND adds a manual CV-properties review (Adobe Reader / Preview) to catch tags exiftool's allowlist permits. |
| T-07-02 | Information Disclosure | Screenshot leaks IPs / hostnames / employer data / live CTF flags | mitigate (manual) | Items 5-12 of the checklist explicitly enumerate forbidden screenshot contents; redaction must be solid black box, not blur (blur is reversible). Phase 3 is when this becomes load-bearing — the checklist is the gate. |
| T-07-03 | Information Disclosure | Sourcemaps reveal full file paths and original variable names | mitigate | `vite.config.ts` `build.sourcemap: false` set in Plan 01; checklist item under Source / Bundle re-verifies on every release. |
| T-07-04 | Information Disclosure | Personal phone / home address / full DOB published | mitigate | Personal Information section of checklist forbids all three; manual gate. |
| T-07-05 | Information Disclosure | Real email harvested by spam scrapers | mitigate | Plan 03's `obfuscate.ts` + Plan 06's JSON-LD email omission; this plan's checklist runs `grep` on `dist/index.html` to verify no plaintext email string slipped through. |
| T-07-06 | Information Disclosure | Console.log / debug helpers shipped to prod | mitigate | Source / Bundle section of checklist runs `grep -rE "console\.(log\|debug)"` on `dist/assets/`. |
| T-07-07 | Information Disclosure / Tabnabbing | External links leak referrer / opener to attacker | mitigate | External Links section runs the `target="_blank"` + `noopener noreferrer` grep. |
| T-07-08 | Information Disclosure | Home-lab diagrams reveal real ISP / router model / RFC1918 internals | accept-for-Phase-1 / mitigate-Phase-3 | Home-Lab section of checklist forbids real IPs/hostnames in any future diagram; flagged as Phase-3-relevant. |
| T-07-09 | Repudiation | Audit run not documented; future Eren can't tell which release the checklist was last run against | mitigate | Each release-time run produces / updates `01-07-SUMMARY.md` with commit SHA + date + per-item PASS/FAIL/DEFERRED. SUMMARY is the audit trail. |

**Block-on-`high`:** all listed threats are mitigated to `medium` or below by the combination of Plan 02's CI gate + this plan's manual checklist + Plan 03/06's source-level mitigations. No `high` severity threats remain unaddressed.
