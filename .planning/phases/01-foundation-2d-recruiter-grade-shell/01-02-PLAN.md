    ```yaml
    # .github/workflows/deploy.yml
    # Source: 01-RESEARCH.md Pattern 11; 01-CONTEXT.md D-15, D-16, D-17
    # Deploys the static Vite bundle to GitHub Pages on every push to main.
    # Action versions verified live 2026-05-06.

    name: Deploy to GitHub Pages

    on:
      push:
        branches: [main]
      workflow_dispatch:

    permissions:
      contents: read
      pages: write
      id-token: write

    # Allow only one concurrent deployment; do not cancel an in-progress run.
    concurrency:
      group: pages
      cancel-in-progress: false

    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Setup Node
            uses: actions/setup-node@v6
            with:
              node-version: '22'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          # Phase 1 CI gates — all blocking, all run BEFORE build (D-16).
          - name: TypeScript typecheck
            run: npx tsc --noEmit

          - name: ESLint (warnings are errors)
            run: npx eslint . --max-warnings=0

          - name: Prettier check
            run: npx prettier --check .

          # Phase 1 test surface (Plan 03's lib tests). Empty until Plan 03 lands;
          # vitest run on no test files exits 0, so this is safe in Wave 2.
          - name: Vitest (utility tests)
            run: npm test

          # OPSEC gate (D-17 / OPS-01) — strip + verify metadata in public/assets/.
          - name: Install exiftool
            run: |
              sudo apt-get update
              sudo apt-get install -y libimage-exiftool-perl

          - name: Strip metadata from public/assets
            run: |
              # If public/assets/ does not exist yet (Plan 02 ships before Plan 05's
              # CV PDF and Plan 06's og-image.png) the loop is a no-op. Once those
              # plans land the strip becomes meaningful. Build does not fail on an
              # empty directory.
              if [ -d public/assets ] && [ "$(ls -A public/assets 2>/dev/null)" ]; then
                exiftool -all= -P -overwrite_original public/assets/*
              else
                echo "public/assets/ is empty or absent — strip step skipped (will activate once content lands)."
              fi

          - name: Verify no metadata remains (fail build if any present)
            run: |
              # Allowed system tags only. Anything else (Author/GPS*/CreatorTool/Software/etc.) fails.
              if [ -d public/assets ] && [ "$(ls -A public/assets 2>/dev/null)" ]; then
                for f in public/assets/*; do
                  [ -f "$f" ] || continue
                  offending=$(exiftool -S \
                    -ExifToolVersion= -FileSize= -FileType= \
                    -FileTypeExtension= -MIMEType= -Directory= -FileName= \
                    -FilePermissions= -FileModifyDate= -FileAccessDate= \
                    -FileInodeChangeDate= "$f")
                  if [ -n "$offending" ]; then
                    echo "OPSEC FAIL: $f still contains metadata:"
                    echo "$offending"
                    exit 1
                  fi
                done
              else
                echo "public/assets/ is empty or absent — verify step has nothing to check."
              fi

          - name: Build (Vite)
            run: npm run build

          - name: Copy index.html → 404.html (SPA fallback for GH Pages deep links)
            run: cp dist/index.html dist/404.html

          - name: Setup Pages
            uses: actions/configure-pages@v6

          - name: Upload artifact
            uses: actions/upload-pages-artifact@v5
            with:
              path: ./dist

      deploy:
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        steps:
          - name: Deploy to GitHub Pages
            id: deployment
            uses: actions/deploy-pages@v5
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner ↔ Source | CI executes against committed source on a sandboxed Ubuntu runner; no inbound network from the runner to private resources. |
| GH Pages CDN ↔ Browser | Static files served over HTTPS; no server-side logic. |
| `public/assets/*` ↔ Public web | Every byte under `public/assets/` becomes publicly downloadable; OPSEC gate must catch metadata leaks before upload. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | EXIF/XMP/GPS/Author/CreatorTool metadata in `public/assets/*` (CV PDF, OG image, screenshots) | mitigate | `exiftool -all= -P -overwrite_original` strip step + offending-tag verify step in workflow. Build fails if any non-allowlisted tag remains. (D-17, OPS-01.) |
| T-02-02 | Tampering | Pinned action versions become deprecated mid-flight | mitigate | All `actions/*` versions verified live 2026-05-06 (RESEARCH.md). Plan 07 OPSEC checklist re-verifies on first real deploy; SUMMARY documents any forced upgrade. |
| T-02-03 | Tampering | Deploy workflow runs on a fork's PR (could leak secrets) | accept | This Phase 1 workflow has no secrets (no `secrets.*` references). The `id-token: write` permission is for OIDC to GH Pages only, not for arbitrary cloud auth. PRs from forks would still need write permissions to deploy, which fork PRs do not have by default. |
| T-02-04 | Elevation of Privilege | Compromised dependency runs in CI | mitigate (partial) | `npm ci` uses pinned `package-lock.json` from Plan 01. Plan 07's OPSEC checklist runs `npm audit --production` as a manual gate. Phase 4 polish may add `npm audit` to the workflow. |
| T-02-05 | Repudiation | Deploy fails silently / no audit trail | accept | GitHub Actions logs every step; deploy URL is surfaced as `${{ steps.deployment.outputs.page_url }}` and visible in the Actions UI. |
| T-02-06 | Information Disclosure | Source maps in `dist/` reveal file paths | mitigate | Plan 01's `vite.config.ts` already sets `build.sourcemap: false`. This plan's workflow runs `npm run build` which honours that config. |
| T-02-07 | Denial of Service | Concurrent deploys race | mitigate | `concurrency: { group: pages, cancel-in-progress: false }` serializes deploys. |
</threat_model>
