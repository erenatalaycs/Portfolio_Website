<key_decisions>
- **Honeypot `botcheck` (CONTEXT D-11)** — `<input type="text" name="botcheck" tabIndex={-1} autoComplete="off">` inside `<div style={{ display: 'none' }} aria-hidden="true">`. `display: none` removes from a11y tree. UI-SPEC pins `type="text"` (Web3Forms canonical example uses checkbox; both work because Web3Forms only checks "non-empty").
- **Silent honeypot abort (CONTEXT D-11 + RESEARCH Pattern 8)** — `botcheck` non-empty → re-render success state WITHOUT calling fetch. Saves Web3Forms quota; bot sees 200-equivalent UX → marks form compromised → doesn't probe further.
- **HTML5 validation only (CONTEXT D-11)** — `required` + `pattern=".+@.+\..+"` + `maxLength`. NO zod / NO react-hook-form / NO vee-validate. Inline `# invalid email` shows on `onBlur`, NOT `onChange` (UI-SPEC: typing-while-validating reads as nagging).
- **Submit `[Sending…]` label** — literal U+2026 horizontal ellipsis (NOT three periods). Submit button + fields disabled during submit so values preserved across retries.
- **Failure copy generic (Pitfall 4)** — `Network error.` UI-only; `submitContact`'s `diagnosticForLog` for `import.meta.env.DEV` console only.
- **Email-reveal in failure state (UI-SPEC)** — reuse `<EmailReveal encoded={identity.emailEncoded} />` verbatim. Same obfuscation defense.
- **Subject pre-fill** — `[Portfolio enquiry] from <Name>` interpolated client-side; falls back to `(name empty)` if name missing.
- **CSP extension** — ADD `https://api.web3forms.com` to `connect-src` (for `fetch` POST) AND `form-action` (defensive for non-fetch fallback).
- **`.env.example` committed** — documents env vars; `.env.local` author-side and gitignored.
- **DO NOT** install web3forms SDK (none exists), hCaptcha (CONTEXT D-11 chose honeypot — UI-SPEC § Anti-Pattern Audit explicitly rejects), or any form library.
</key_decisions>
