# BLOCKING CONSTRAINTS — Read Before Anything Else

- [ ] CONSTRAINT: **drei `<Html transform distanceFactor>` formula is empirical** — the docs and Phase 3 spec say DF range 1.5–2.5; this is THEORETICAL and was never deploy-verified. Real behaviour: world_size ≈ CSS_px × DF / 400 (matrix elements multiplied by DF/400). For 600 px CSS to fit on a 0.55 m monitor screen, DF ≈ 0.37. **Verify visually via screenshot before committing any DF value.**
- [ ] CONSTRAINT: **Canvas `frameloop="demand"` in R3F v9 + React 19 does NOT auto-paint on first scene-tree commit.** Without explicit `invalidate()` in `onCreated`, canvas stays black forever waiting for user input. FocusController GSAP animations also need `invalidate()` in `onUpdate` — otherwise camera state mutates but no frame renders.
- [ ] CONSTRAINT: **CSP must allow `worker-src 'self' blob:`, `script-src ... 'wasm-unsafe-eval'`, `connect-src ... blob:`, `font-src 'self' data:`, `img-src 'self' data: blob:`** for R3F + drei + useGLTF + @fontsource. The default `'self'`-only directives silently break Draco workers, WebAssembly instantiate, GLB texture blobs, and inline data: fonts.
- [ ] CONSTRAINT: **Real Poly Haven GLB asset cannot be uniformly scaled to fit the Phase 2 procedural envelope.** GLB ratio 2.0:0.79 ≠ procedural ratio 1.2:0.79. Plan 04-06's real-GLB Path A is fragile — Phase 2 procedural geometry is the known-good baseline.

**Do not proceed until all four boxes are checked.**
