workstation/ — CC0 GLB composite provenance
=============================================

All assets in this directory are licensed CC0 1.0 Universal (Public Domain
Dedication). No attribution is required, but it is recorded here for
provenance per Phase 4 CONTEXT D-01 + OPSEC posture (Plan 04-06).

Pipeline: every asset passed through `gltfjsx@~6.5 --transform` (which
bundles draco geometry compression + texture resize + webp recompress +
dedup + prune). The transform rebuilds the GLB binary from raw geometry
+ raw textures, so any source-side metadata (author, software, hostnames
embedded in JPEG EXIF, etc.) is discarded at pipeline time. exiftool
post-pass confirmed no author / software / copyright / GPS / artist /
creator metadata detectable in the output GLBs.

Monitor mesh
------------
NOT shipped as a CC0 GLB. The Poly Haven catalogue (verified 2026-05-11 via
`https://api.polyhaven.com/assets?type=models` keyword search for "monitor",
"screen", "tv", "computer") contains NO CC0 monitor / display model — only
`chinese_screen_panels` (a folding privacy screen, not a computer display).

The Phase 4 deviation is to retain the existing **procedural monitor geometry**
already shipped inside `src/scene/Monitor.tsx` (frame box + emissive screen
plane + stand cylinder, Phase 2 D-06). This preserves:
  - The Phase 3 click-to-focus screen plane that drei `<Html transform>`
    projects DOM content onto (the geometry-coupled invariant).
  - The Phase 4 D-08 Bloom tuning (screen emissive uses --color-accent token).

The monitor wrapper geometry remains procedural; the desk, lamp, and bookshelf
ship as CC0 GLBs. The scene composition is therefore a "CC0 + procedural
composite", not a pure CC0 composite. Acknowledged deviation from the Plan
04-06 file-list (no monitor.glb shipped); documented in 04-06-SUMMARY.md.
